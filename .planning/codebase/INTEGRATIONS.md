# External Integrations

**Analysis Date:** 2026-05-04

## APIs & External Services

**Social Media Metrics:**
- YouTube Data API v3 - Channel search, subscriber counts, recent videos
  - SDK/Client: Native `urllib` (Python)
  - Auth: `YOUTUBE_API_KEY` env var
  - Implementation: `scrapers/social_scraper.py` lines 46–200+

- Thunderbit AI Extraction - Facebook, Instagram, X (primary)
  - SDK/Client: `requests` library
  - Auth: Bearer token via `THUNDERBIT_API_KEY` env var
  - Endpoint: `https://openapi.thunderbit.com/openapi/v1/extract`
  - Implementation: `scrapers/social_scraper.py` lines 125–165
  - Schema format: Simple key-value descriptions (not JSON Schema)
  - Retry logic: Up to 2 attempts with 5-second backoff on failure

- ScraperAPI - Fallback social scraper (Facebook, Instagram, X)
  - SDK/Client: `requests` library
  - Auth: API key via query parameter
  - Endpoint: `http://api.scraperapi.com`
  - Implementation: `scrapers/social_scraper.py` lines 76–99
  - Supports rendering, premium mode, country-code geofencing

**AI Analysis:**
- Anthropic Claude API (claude-sonnet-4-6)
  - SDK/Client: `anthropic` Python package v0.40.0
  - Auth: `ANTHROPIC_API_KEY` env var
  - Use cases:
    - Pricing page extraction (`scrapers/pricing_scraper.py`)
    - AI-driven competitive intelligence synthesis (`scrapers/ai_analyzer.py`)
    - Tool-use for structured outputs (portfolio recommendations)
  - Implementation: `scrapers/ai_analyzer.py` lines 1–150+, `scrapers/pricing_scraper.py` lines 54–100+

**RSS/Feeds:**
- Generic RSS/Atom feeds - News scraping
  - SDK/Client: `feedparser` v6.0.11
  - Implementation: `scrapers/news_scraper.py`
  - Parses competitor news, blog feeds

**Website Scraping:**
- Playwright v1.49.0 - Browser automation for JavaScript-heavy sites
  - Use cases:
    - Pricing pages: `scrapers/pricing_scraper.py`
    - Reputation sites (Trustpilot, ForexPeaceArmy, App Store): `scrapers/reputation_scraper.py`
    - Account types, promotions: `scrapers/account_types_scraper.py`, `scrapers/promo_scraper.py`
  - Supports async/await, wait selectors, timeout handling

## Data Storage

**Databases:**
- SQLite (embedded)
  - Connection: File-based at `data/competitor-intel.db`
  - Client: better-sqlite3 (Node.js), sqlite3 (Python)
  - ORM: Drizzle ORM (TypeScript) for type-safe queries
  - Tables managed by `src/db/schema.ts`

**File Storage:**
- Local filesystem only (no cloud object storage)
  - Database: `data/competitor-intel.db`
  - Backup: `data/competitors.db` (legacy)
  - Logs: `logs/` directory (generated by scrapers)

**Caching:**
- None (in-memory rate limiter for login auth only)
  - `src/app/api/auth/login/route.ts` maintains in-memory failed attempt tracking (per-IP, 15-min reset)

## Authentication & Identity

**Dashboard Access:**
- Custom password-based auth (no third-party provider)
  - Implementation: `src/app/api/auth/login/route.ts`
  - Method: SHA-256 hash stored in `DASHBOARD_PASSWORD` env var
  - Session: HTTP-only cookie with 8-hour max age
  - Security: Constant-time comparison, rate limiting (5 failed attempts per IP per 15 minutes)

**API Access:**
- No authentication for internal API routes (`:8080/api/*`)
- MCP server access: Assumes trusted local client (`mcp-server/`)

## Monitoring & Observability

**Error Tracking:**
- None (no external service — errors logged to stdout/stderr)

**Logs:**
- Stdout/stderr from scraper processes
- Log files generated in `logs/` directory by Python scrapers
- Change detection logged to `change_events` table in SQLite

**Scraper Status:**
- Tracked in `scraper_runs` table (SQLite)
  - Stores success/failure, run timestamp, error messages
  - Queried by admin dashboard to detect stale data

## Security Headers & Policies

**Content Security Policy (CSP):**
- Strict: `default-src 'self'`
- Script: `'self' 'unsafe-inline'` + `'unsafe-eval'` in dev only
- Style: `'self' 'unsafe-inline' https://fonts.googleapis.com`
- Fonts: `'self' https://fonts.gstatic.com`
- Images: `'self' data: blob:`
- Connections: `'self'` only
- Frame ancestors: `'none'`

**Other Security Headers:**
- `X-Frame-Options: DENY` - Clickjacking protection
- `X-Content-Type-Options: nosniff` - MIME sniffing prevention
- `Referrer-Policy: strict-origin-when-cross-origin` - Referrer leakage prevention
- `Permissions-Policy: camera=(), microphone=(), geolocation=()` - Disable unneeded browser features
