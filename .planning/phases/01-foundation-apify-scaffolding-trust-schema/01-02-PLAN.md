**`SecretRedactionFilter(logging.Filter)`** class:
- `__init__`: Snapshot env-var values at construction. Skip values shorter than 6 chars (avoid trivial substring match). Also pick up any env var starting with `_SECRET_ENV_PREFIX`.
- `filter(record)`: Replace each captured secret in `record.getMessage()` with `[REDACTED]`. Apply each `_TOKEN_PATTERNS` regex. If `record.msg` was modified, set `record.msg = redacted` and clear `record.args = ()` to avoid double-format errors. Always return True (don't drop records).

**`install_redaction()`** function:
- Attach a `SecretRedactionFilter` to the root logger.
- Idempotent: if root logger already has a `SecretRedactionFilter` in its filters list, do nothing.

| Application code → log handler (file / stdout) | Untrusted log content (URLs, error messages including request bodies) flows here; secrets must be stripped first |
