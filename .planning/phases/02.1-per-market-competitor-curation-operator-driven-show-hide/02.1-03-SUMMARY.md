## Decisions Made

- **Manual `split(",")` over CSV library** — serp_market_research.py emits no quoted fields; ~10-line parser with explicit column-name lookups (`competitor_id`, `queries_appeared`, `best_rank`) is cheaper than a new dependency and matches D2.1-13 cost rationale.
- **Filter inserted after `.sort()` (not fused into the existing `.filter()` chain)** — observationally equivalent to filtering before the sort, but keeps `socialRows` (Phase 2 output) and `filteredSocialRows` (Phase 2.1 output) as visibly separate consts so the layered relationship is immediately obvious to future readers.
- **Generic EmptyState message reused** — the existing "No social data yet for this market" copy covers both the "no data flowing" case and the "no active curation" case; plan-allowed friendlier nudge ("Use the admin panel to mark active") deferred because the generic message is correct for both and the admin UI itself (Plan 02.1-04) ships in the same wave.
- **Sparkles + amber "Watching" badge for the Emerging rail** — intentionally distinct from the green "Market" / gray "Global" DataSourceBadge palette in the Digital Presence section above; different concept (uncurated watchlist vs. data-source provenance), different visual signal.
- **`emergingRail` awaited serially after the main Promise.all** — CSV read is local fs and fast; folding it into the existing Promise.all would require restructuring the destructuring tuple shape with no meaningful latency benefit.
- **SERP-discovered competitor IDs not in `scrapers/config.py` are silently skipped** — per plan, defer to operator to add the broker to config first (D2.1-13 cost rationale: not auto-onboarding new brokers from SERP alone).
